Feedback Tool Integration Guide

This guide explains how to integrate and troubleshoot the web application feedback tool with the Mashaaer Feelings application.

Note: We have encountered persistent challenges with the feedback tool reporting that the web server is unreachable despite implementing industry best practices for web accessibility and CORS configuration.

Health Check Endpoints

GET /health

Ultra-minimal health check endpoint at root level

curl -v http://localhost:5000/health

GET /api/status

Get comprehensive system status information

curl -v http://localhost:5000/api/status

GET /api/ping

Simple ping endpoint for connectivity testing

curl -v http://localhost:5000/api/ping

CORS Configuration

All API endpoints are configured with the following CORS headers:

Access-Control-Allow-Origin: <dynamic based on request origin or '*'>
Access-Control-Allow-Methods: GET, POST, OPTIONS
Access-Control-Allow-Headers: *
Access-Control-Max-Age: 3600

The application dynamically sets the Access-Control-Allow-Origin header based on the incoming request's Origin header, with fallback to * for maximum compatibility.

Testing with Curl

Basic Health Check

curl -v http://localhost:5000/health

Test with Custom Origin

curl -H "Origin: https://example.com" -v http://localhost:5000/api/status

Test OPTIONS Preflight

curl -X OPTIONS -H "Origin: https://example.com" \
  -H "Access-Control-Request-Method: POST" \
  -H "Access-Control-Request-Headers: content-type" \
  -v http://localhost:5000/api/status

Troubleshooting

If you're experiencing issues with the feedback tool, try the following steps:

  1. Verify direct browser access to the application at /
  2. Check the ultra-minimal endpoints like /health and /api/ping
  3. Use the CORS Diagnostic tool to check for CORS issues
  4. Check server logs for detailed information about requests and responses
  5. Try the standalone minimal server at /standalone_minimal_server.py
Important: If you've verified that all endpoints are accessible via direct browser access and curl commands but the feedback tool still reports the server as unreachable, it might be a compatibility issue specific to the feedback tool rather than a problem with the server implementation.
